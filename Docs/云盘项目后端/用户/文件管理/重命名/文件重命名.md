# 文件重命名

- 请求地址：/repository/file/rename

- 请求方式：PUT

- 请求体编码：text/json

- 响应体编码：text/json

- 请求头：

  ```
  Authorization:登录状态Token(string)
  ```

- 请求参数：

  ```json
  {
      repositoryId = 仓库ID(int)
  	oldName = 原文件名(string)
  	newName = 新文件名(string)
  	path = 目标路径(string)
  }
  ```

- 返回参数：

  ```json
  {
      code:状态码(int) {1 成功，0 失败，-1 无访问权限，-2 获取从仓库信息失败，-3 目标路径不存在，-4 原文件不存在，-5 目标文件已存在，-6 文件名不合法}
      message:状态说明(string) {成功，失败，无访问权限，获取从仓库信息失败，目标路径不存在，原文件不存在，目标文件已存在，文件名不合法}
  	data:{
          repository:{
              id = 仓库ID(int)
  			userId = 用户ID(int)
              status = 仓库状态(int)
              useSize = 已用空间大小(long)byte
              repoSize = 仓库总空间大小;(long)byte
              folder = 文件根目录(object)
              recycleBin = 回收站目录(object)
          }
      }
  }
  ```

- 流程图：

  ```mermaid
  graph TD
  start((开始))-->Token解析-->TokenID-->j1{查询用户信息}--失败-->返回无修改权限-->finish((结束))
  j1--成功-->j2{比对仓库ID}--不相同-->返回无修改权限
  j2--相同-->j3{查询仓库信息}--失败-->返回获取仓库信息失败-->finish
  j3--成功-->j4{查找目标路径}--失败-->返回目标路径不存在-->finish
  j4--成功-->j5{查找原文件}--失败-->返回原文件不存在-->finish
  j5--成功-->j6{判断是否重名}--失败-->返回目标文件已存在-->finish
  j6--成功-->获取原文件信息-->创建新文件信息-->插入仓库信息-->删除原文件信息-->保存仓库信息-->JSON封装-->返回仓库信息-->finish
  start-->JSON解析
  JSON解析-->仓库ID-->j2
  JSON解析-->目标路径-->j4
  JSON解析-->原文件名-->j5
  新文件名-->j7{文件名有效性检验}--失败-->返回文件名不合法-->finish
  j7--成功-->j1
  JSON解析-->新文件名-->j6
  
  ```
